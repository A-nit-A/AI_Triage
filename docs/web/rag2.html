Concepto Avanzado

Arquitectura RAG 2.0

Más allá de la simple recuperación, el RAG Agéntico permite que el sistema razone sobre la calidad del contexto, corrija sus errores y decida dinámicamente qué herramienta usar.

RAG 1.0 (Lineal)

  • Confianza ciega en la primera búsqueda vectorial.
  • Propenso a alucinaciones si el contexto es pobre.
  • No entiende preguntas complejas o ambiguas.

RAG 2.0 (Agéntico)

  • Crítica y evaluación de la relevancia del contexto.
  • Reescritura inteligente de consultas (Query Translation).
  • Combinación de búsqueda semántica y de palabras clave.

Componentes del Agente RAG

1. Re-escritura (Query Translation)

El usuario pregunta: *"¿Mi paquete?"*. El agente traduce: *"¿Cuál es la política de tiempos de entrega y estado de envíos?"*. Esto optimiza radicalmente la precisión de ChromaDB. 

# Generar versiones optimizadas para búsqueda
template = "Reescribe esta pregunta para búsqueda vectorial: {pregunta}"
rewrite_chain = PromptTemplate(template) | llm | StrOutputParser()

queries = rewrite_chain.invoke("¿Cuánto tardan?")
# Evaluación de Relevancia (Self-RAG)
def check_relevance(pregunta, contexto):
    score = grader_llm.invoke(f"Q: {pregunta}, C: {contexto}")
    if score == 'irrelevante':
        return rewrite_and_retry() 
    return generate_answer()

2. Auto-Grader (Verificación)

Un LLM secundario actúa como "juez", analizando si los fragmentos recuperados realmente sirven para responder. Si no sirven, el agente no inventa; vuelve a buscar o escala.

3. Recuperación Híbrida

Combina lo mejor de dos mundos: Vectores (significado) + BM25 (exactitud). Crucial para encontrar códigos de error o nombres de productos específicos. 

# Ensemble Retriever (LangChain)
hybrid_retriever = EnsembleRetriever(
    retrievers=[chroma_retriever, bm25_retriever],
    weights=[0.7, 0.3] # Prioriza semántica pero usa exactitud
)
