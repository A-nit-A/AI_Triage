AI TRIAGE

Construcción de un Agente de Triaje de Emails inteligente para TechPyme usando RAG y modelos locales.

El Problema de TechPyme

Una pequeña tienda online recibe decenas de correos al día. El dueño pierde 2 horas diarias respondiendo dudas básicas o derivando mensajes al departamento correcto manualmente.

Pérdida de Tiempo

Tareas repetitivas que no aportan valor estratégico al negocio.

Respuesta Lenta

El cliente espera demasiado tiempo por una duda sencilla de resolver.

Error Humano

Correos mal derivados o respuestas con información desactualizada.

Objetivo de la Actividad

Programar un agente inteligente que actúe como filtro inicial, respondiendo automáticamente usando las FAQs de la empresa o escalando a un humano si es necesario.

Modelo de IA Local (Ollama)

Utilizaremos Ollama para ejecutar modelos potentes directamente en local.

  • Privacidad: Los datos de los clientes nunca salen de tu ordenador.
  • Cero Costes: No pagamos por tokens a APIs externas (OpenAI, Claude).

ChromaDB Vector Store

Carga y Fragmentación

from langchain_community.document_loaders import PyPDFLoader
from langchain_text_splitters import RecursiveCharacterTextSplitter

loader = PyPDFLoader("faqs.pdf")
docs = loader.load()

text_splitter = RecursiveCharacterTextSplitter(
    chunk_size=1000, 
    chunk_overlap=200
)
splits = text_splitter.split_documents(docs)

Almacenamiento Vectorial

from langchain_chroma import Chroma
from langchain_community.embeddings import OllamaEmbeddings

vectorstore = Chroma.from_documents(
    documents=splits, 
    embedding=OllamaEmbeddings(model="llama3"),
    persist_directory="./db_faqs"
)

Programación del Agente

El Razonamiento

El agente utiliza un Prompt de Sistema para definir su comportamiento. Es vital indicarle que solo use el contexto proporcionado.

"Eres un agente de soporte de TechPyme. Responde preguntas de clientes usando solo el contexto de las FAQs. Si la información no está presente o el tono del cliente es agresivo, responde exactamente con la etiqueta: [ESCALAR_A_HUMANO]." 
from langchain.chains import RetrievalQA

qa_chain = RetrievalQA.from_chain_type(
    llm=Ollama(model="llama3"),
    retriever=vectorstore.as_retriever(),
    chain_type="stuff"
)

# Ejemplo de triaje
email = "¿Puedo devolver un producto tras 15 días?"
resultado = qa_chain.invoke(email)

if "[ESCALAR_A_HUMANO]" in resultado:
    print("Enviando alerta a Soporte Humano...")
else:
    print(f"Respuesta generada: {resultado}")

Evolución: RAG 2.0

Un sistema básico puede fallar si la pregunta es ambigua. El RAG Agéntico añade capas de razonamiento adicionales.

Auto-Corrección

El agente verifica si su propia respuesta responde realmente a la pregunta antes de enviarla.

Multi-Búsqueda

Genera múltiples versiones de la duda del usuario para encontrar mejores coincidencias en la DB.
